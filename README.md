# Content Filter Utility

Утилита командной строки на Java для фильтрации содержимого файлов. Программа читает входные файлы и разделяет данные на три типа: целые числа, вещественные числа и строки, записывая их в соответствующие выходные файлы.

## Особенности проекта

-   **Java 17+**: Проект настроен для совместимости с современными версиями Java (включая Java 21 и Java 24).
-   **Java Records**: Активное использование неизменяемых записей (Records) для лаконичных моделей данных.
-   **Lombok**: Использование аннотаций `@Slf4j`, `@Getter`, `@Builder` для сокращения бойлерплейта.
-   **Clean Code**: Соблюдение стандартов кодирования (Checkstyle, SpotBugs).
-   **Паттерны проектирования**:
    -   *Visitor* для записи файлов и сбора статистики.
    -   *Builder* для конфигурации.
    -   *Strategy* для парсинга данных.

## Требования

-   **JDK 17** или выше.
-   **Maven 3.8+**.

## Сборка

Проект использует Maven для сборки и управления зависимостями.

```bash
mvn clean verify
```

Эта команда:
1.  Очистит директорию `target`.
2.  Скомпилирует код.
3.  Запустит тесты.
4.  Проверит стиль кода (Checkstyle) и найдет потенциальные баги (SpotBugs).
5.  Соберет исполняемый jar со всеми зависимостями.

## Запуск

После успешной сборки исполняемый файл будет находиться в директории `target`.

```bash
java -jar target/content-filter-1.0-SNAPSHOT.jar [options] <files...>
```

### Опции командной строки

-   `-o <path>` : Указать путь для выходных файлов (по умолчанию текущая директория).
-   `-p <prefix>` : Добавить префикс к именам выходных файлов.
-   `-a` : Режим добавления в существующие файлы (по умолчанию файлы перезаписываются).
-   `-s` : Вывести краткую статистику (количество элементов).
-   `-f` : Вывести полную статистику (мин, макс, сумма, среднее).

### Примеры использования

1.  **Простая фильтрация**:
    ```bash
    java -jar target/content-filter-1.0-SNAPSHOT.jar input.txt
    ```

2.  **С сохранением в папку `results`, префиксом `filtered_` и полной статистикой**:
    ```bash
    java -jar target/content-filter-1.0-SNAPSHOT.jar -o results -p filtered_ -f input1.txt input2.txt
    ```

3.  **Режим добавления данных (Append)**:
    ```bash
    java -jar target/content-filter-1.0-SNAPSHOT.jar -a input.txt
    ```

## Разработка

В проекте настроены инструменты статического анализа, которые запускаются автоматически при сборке:
-   **Checkstyle**: проверяет форматирование и стиль кода.
-   **SpotBugs**: ищет потенциальные ошибки и уязвимости.

Для подавления ложных срабатываний Checkstyle используется файл `checkstyle-suppressions.xml`.
